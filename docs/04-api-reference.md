# API Reference

The backend exposes a REST API built with FastAPI. Interactive docs are available at `/docs` when running locally.

**Base URL (local):** `http://localhost:8000`
**Base URL (production):** `https://3gpp.deltatechus.com`

---

## Endpoints

### `GET /health`

Returns the service status and number of chunks indexed in ChromaDB.

**Response**
```json
{
  "status": "ok",
  "chunks_indexed": 445000
}
```

| Field | Type | Description |
|---|---|---|
| `status` | string | `"ok"` or `"degraded"` |
| `chunks_indexed` | integer | Total chunks in ChromaDB |

---

### `POST /api/chat`

Synchronous chat endpoint. Returns the full answer once generation is complete.

**Request Body**
```json
{
  "question": "What is 5G NR numerology?",
  "release_filter": "Rel-18",
  "series_filter": "38_series"
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `question` | string | Yes | The user's question |
| `release_filter` | string | No | Filter results to a specific release (e.g. `"Rel-18"`) |
| `series_filter` | string | No | Filter results to a specific series (e.g. `"38_series"`) |

**Response**
```json
{
  "answer": "5G NR numerology refers to...",
  "sources": [
    {
      "spec_name": "38211-i40",
      "spec_number": "38211",
      "release": "Rel-18",
      "series": "38_series",
      "score": 0.87
    }
  ],
  "sub_queries": [
    "5G NR numerology definition",
    "subcarrier spacing options NR",
    "numerology parameters TS 38.211"
  ]
}
```

| Field | Type | Description |
|---|---|---|
| `answer` | string | Markdown-formatted answer |
| `sources` | array | List of cited 3GPP specs |
| `sources[].spec_name` | string | Full spec filename |
| `sources[].spec_number` | string | TS number (e.g. `"38211"`) |
| `sources[].release` | string | 3GPP release |
| `sources[].series` | string | Spec series |
| `sources[].score` | float | Cosine similarity score (0–1) |
| `sub_queries` | array | Sub-queries generated by the Query Planner |

### `POST /api/chat/stream`

Streaming chat endpoint using **Server-Sent Events (SSE)**. Returns tokens as they are generated, followed by a final event with sources.

**Request Body** — same as `/api/chat`

**Response** — `text/event-stream`

The stream emits a sequence of SSE events:

#### Token events (during generation)
```
data: {"type": "token", "content": "5G"}

data: {"type": "token", "content": " NR"}

data: {"type": "token", "content": " numerology"}
```

#### Done event (end of stream)
```
data: {"type": "done", "sources": [...], "sub_queries": [...]}
```

**JavaScript client example:**
```javascript
const response = await fetch('/api/chat/stream', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ question: 'What is beam management?' })
});

const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;

  const lines = decoder.decode(value).split('\n');
  for (const line of lines) {
    if (!line.startsWith('data: ')) continue;
    const event = JSON.parse(line.slice(6));
    if (event.type === 'token') {
      process.stdout.write(event.content);
    } else if (event.type === 'done') {
      console.log('\nSources:', event.sources);
    }
  }
}
```

### `GET /`

Root endpoint — returns service info.

**Response**
```json
{
  "name": "3gppSpec 3GPP AI Assistant",
  "status": "running",
  "docs": "/docs"
}
```

---

## Error Responses

| Status | Meaning |
|---|---|
| `422 Unprocessable Entity` | Missing required field in request body |
| `500 Internal Server Error` | LLM API error or ChromaDB failure |

---

## CORS

The API allows requests from origins configured in `ALLOWED_ORIGINS` environment variable. Default: `http://localhost:3000`.

Production adds `https://3gpp.deltatechus.com`.
